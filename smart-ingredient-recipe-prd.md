## 1. Product Overview

**Smart Ingredient** is a single-screen mobile utility app. The user types 3–4 ingredients they have at home, taps one button, and the app returns a complete, easy-to-follow recipe generated by an LLM — including a recipe title, full ingredient list with quantities, and 8–12 detailed step-by-step cooking instructions.

The app must feel fast, warm, and confident. It should look like a polished consumer cooking app, not a hackathon prototype.

---

## 2. Goals & Success Metrics

| Goal | Metric |
|---|---|
| User enters ingredients and gets a recipe | 100% of valid submissions return a structured recipe |
| Recipe is detailed enough to actually cook | Minimum 7 steps returned per recipe |
| Error states never leave the user confused | Every error has a recovery action |
| UI feels polished and intentional | No default React Native grey, no stock fonts |
| AI response feels fast | Streaming or skeleton loading within 1s of tap |

---

## 3. Target User

**Home cook, 20–40 years old.** Has random ingredients, limited time, no meal plan. Wants one solid suggestion — not a list of options to scroll through. Trusts AI to do the thinking.

---

## 4. Screen Architecture

The entire app lives on **one screen** with three visual states:

```
┌─────────────────────────────┐
│         INPUT STATE         │  ← Default on launch
│  Header + Tag Input + CTA   │
└─────────────────────────────┘

┌─────────────────────────────┐
│        LOADING STATE        │  ← After CTA tap, during API call
│  Animated skeleton / pulse  │
└─────────────────────────────┘

┌─────────────────────────────┐
│        RESULT STATE         │  ← Recipe returned
│  Recipe card, scrollable    │
│  "Try Again" back to input  │
└─────────────────────────────┘
```

There is no navigation. No tabs. No modals (except an inline toast for errors).

## 7. UX Flow & Interactions

### 7.1 Complete Happy Path

```
1. App opens → Input State
   └── Header visible, input empty, CTA disabled (greyed)

2. User taps input field
   └── Keyboard rises
   └── Input border animates to orange
   └── Instruction label fades in: "Separate ingredients with commas"

3. User types: "chicken, broccoli, soy sauce, rice"
   └── Ingredient tags appear live below input
   └── CTA button becomes active (orange)
   └── Counter shows "4 ingredients"

4. User taps "Generate My Recipe"
   └── Button scales down (press feedback)
   └── Keyboard dismisses
   └── Input area fades out
   └── Skeleton loading state fades in
   └── Loading message rotates every 2s

5. API returns recipe (5–10 seconds)
   └── Skeleton fades out
   └── Recipe card slides up from bottom (spring)
   └── Recipe title, ingredients, steps visible
   └── User scrolls to read all steps

6. User taps "Cook Something Else"
   └── Recipe card slides down
   └── Input state fades back in
   └── Input field is cleared
   └── Ready for new ingredients
```

### 7.2 Keyboard Behavior

- `ScrollView` wraps entire screen with `keyboardShouldPersistTaps="handled"`
- When keyboard is open, the CTA button anchors above the keyboard using `KeyboardAvoidingView`
- The button must always be visible — user should not have to dismiss keyboard to tap it

### 7.3 Ingredient Tag Parsing Logic

```js
// Parse on every keystroke
const parseIngredients = (text) => {
  return text
    .split(',')
    .map(item => item.trim())
    .filter(item => item.length > 1); // ignore single chars
};

// Cap at 10 ingredients
// Show warning if fewer than 2: "Add at least 2 ingredients for a recipe"
```

## 8. Edge Cases & Error Handling

### 8.1 Input Validation

| Case | Trigger | UI Response |
|---|---|---|
| Empty input | CTA tapped with blank field | Button stays disabled. Add shake animation to input field. Show inline hint: *"Enter at least 2 ingredients to start."* |
| Only 1 ingredient | 1 tag detected | CTA enabled but shows warning badge. On tap, show toast: *"Try adding 1–2 more ingredients for a better recipe."* Still proceeds. |
| Too many ingredients (10+) | User keeps typing | Input character stops accepting new commas. Counter turns red. Show: *"That's plenty! Using first 10 ingredients."* |
| Non-food input | "asphalt, concrete, engine oil" | API returns a graceful message. Show: *"Hmm, those don't look like food ingredients. Try again?"* Do not show a recipe. |
| Input is all one word | No commas typed | Show inline hint below input: *"Separate ingredients with commas — e.g. eggs, butter, flour"* |
| Duplicate ingredients | "chicken, chicken, rice" | De-duplicate before sending to API. Tags merge. Show no error. |
| Very short ingredient names | "a, b, c" | Filter out tokens under 2 chars before API call. |

### 8.2 API / Network Errors

| Case | Trigger | UI Response |
|---|---|---|
| No internet | `fetch` throws network error | Error toast: *"No internet connection. Check your Wi-Fi and try again."* + retry button in toast. |
| API timeout (>15s) | `AbortController` fires | Dismiss skeleton. Toast: *"This is taking too long. Try again?"* |
| Groq rate limit (429) | API returns 429 | Toast: *"We hit a limit. Please wait a moment and retry."* |
| Groq server error (500) | API returns 500 | Toast: *"Something went wrong on our end. Tap to retry."* |
| Malformed JSON response | JSON.parse fails | Fallback: try to display raw text. If unusable, show toast: *"Couldn't read the recipe. Tap to try again."* |
| Empty response | `content` field empty | Toast: *"No recipe was generated. Try different ingredients."* |
| API key missing | Key undefined/empty | Show a persistent banner (dev mode only): *"Groq API key not configured. See .env setup."* |

### 8.3 Retry Behavior

Every error state (except validation) must offer a **one-tap retry**. Either:
- A `[Retry]` button inside the toast, OR
- Tapping the toast itself retriggers the last API call

Do not require the user to re-enter ingredients to retry. Store `lastIngredients` in state.

### 8.4 Loading Timeout

```js
const controller = new AbortController();
const timeoutId = setTimeout(() => controller.abort(), 15000); // 15s timeout

try {
  const response = await fetch(GROQ_URL, {
    signal: controller.signal,
    ...
  });
  clearTimeout(timeoutId);
} catch (err) {
  if (err.name === 'AbortError') {
    showToast('Taking too long. Try again?');
  } else {
    showToast('Connection error. Check your internet.');
  }
}
```

## 9. AI Integration — Groq API

### 9.1 API Call Setup

```js
// config/groq.js
const GROQ_API_URL = 'https://api.groq.com/openai/v1/chat/completions';
const GROQ_API_KEY = process.env.EXPO_PUBLIC_GROQ_API_KEY;

export const generateRecipe = async (ingredients, signal) => {
  const response = await fetch(GROQ_API_URL, {
    method: 'POST',
    signal,
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${GROQ_API_KEY}`,
    },
    body: JSON.stringify({
      model: 'llama3-70b-8192',   // Best Groq model for instruction-following
      temperature: 0.7,
      max_tokens: 1500,           // Enough for 10+ detailed steps
      response_format: { type: 'json_object' },
      messages: [
        { role: 'system', content: SYSTEM_PROMPT },
        { role: 'user', content: buildUserPrompt(ingredients) }
      ],
    }),
  });

  if (!response.ok) {
    throw new GroqAPIError(response.status, await response.text());
  }

  const data = await response.json();
  const raw = data.choices[0]?.message?.content;
  return JSON.parse(raw);
};
```

### 9.2 Model Choice

Use `llama3-70b-8192`. It is:
- Available on Groq's free tier
- Excellent at structured JSON output
- Detailed and coherent for recipe tasks
- Significantly better than `llama3-8b` for multi-step instructions

Fallback: `mixtral-8x7b-32768` if llama3-70b is unavailable.

### 9.3 Expected JSON Schema

```json
{
  "title": "Teriyaki Chicken with Broccoli and Steamed Rice",
  "cook_time": "25 minutes",
  "servings": "2",
  "difficulty": "Easy",
  "ingredients": [
    { "quantity": "2", "unit": "pieces", "name": "chicken breast, thinly sliced" },
    { "quantity": "1", "unit": "head", "name": "broccoli, cut into florets" },
    { "quantity": "3", "unit": "tbsp", "name": "soy sauce" },
    { "quantity": "1", "unit": "cup", "name": "jasmine rice" },
    { "quantity": "1.5", "unit": "cups", "name": "water (for rice)" }
  ],
  "steps": [
    {
      "step_number": 1,
      "title": "Prepare the rice",
      "instruction": "Rinse the jasmine rice under cold running water 2–3 times until the water runs mostly clear. This removes excess starch and prevents the rice from becoming sticky. Drain well and set aside."
    },
    {
      "step_number": 2,
      "title": "Cook the rice",
      "instruction": "In a medium saucepan, combine the rinsed rice and 1.5 cups of cold water. Bring to a boil over high heat, then immediately reduce to the lowest heat setting. Cover tightly and cook for 12 minutes. Do not lift the lid during cooking."
    }
    // ... 7–10 more steps
  ],
  "tips": "For extra flavour, marinate the chicken in soy sauce for 15 minutes before cooking."
}
```

## 10. Prompt Engineering

### 10.1 System Prompt

```
You are a professional chef and recipe writer. Your job is to create clear, 
detailed, easy-to-follow recipes based on a list of ingredients a home cook 
has available.

STRICT RULES:
1. Only use the ingredients provided. You may add water, salt, pepper, and 
   basic cooking oil without being asked — these are pantry staples.
2. Return ONLY valid JSON. No markdown, no preamble, no explanation outside 
   the JSON object.
3. Write a minimum of 8 steps and a maximum of 12 steps.
4. Each step must be 2–4 sentences long. Explain the "why" behind each 
   action — not just "do this" but "do this because...". This helps beginner 
   cooks understand the process.
5. Include a clear step title for each step (e.g., "Season the chicken").
6. Include realistic quantities for each ingredient, not vague amounts.
7. The recipe must be realistically cookable in a standard home kitchen.
8. If the ingredients cannot form a real recipe (e.g., non-food items), 
   return: { "error": "These don't appear to be food ingredients." }
9. Always include a "tips" field with one useful cooking tip.
10. Estimate cook_time honestly. Do not say "10 minutes" if it takes 30.
```

### 10.2 User Prompt Builder

```js
const buildUserPrompt = (ingredients) => {
  const list = ingredients.join(', ');
  return `Create a complete recipe using these ingredients: ${list}.
  
  Follow the JSON schema exactly:
  {
    "title": string,
    "cook_time": string,
    "servings": string,
    "difficulty": "Easy" | "Medium" | "Hard",
    "ingredients": [{ "quantity": string, "unit": string, "name": string }],
    "steps": [{ "step_number": number, "title": string, "instruction": string }],
    "tips": string
  }`;
};
```

### 10.3 Why This Prompt Works

- **"Explain the why"** forces multi-sentence steps, not one-liners
- **`response_format: json_object`** on Groq forces strict JSON output
- **Minimum 8 steps** is explicit — LLMs compress without this constraint
- **Quantities required** prevents vague instructions like "some" or "a bit"
- **Error fallback** handles non-food ingredients gracefully

## 11. Recipe Display Requirements

### 11.1 Step Requirements (Non-Negotiable)

Each generated step must:
- Have a step number (①, ②… or 1., 2.)
- Have a short step title (bold, 14sp)
- Have 2–5 sentences of instruction body (15sp, regular)
- Be visually separated from the next step (16px gap + optional thin divider)

### 11.2 Ingredient Display

- Quantity + unit bolded: **2 tbsp** soy sauce
- Ingredient name in regular weight
- Bullet point in `--accent-secondary` color
- One ingredient per line

### 11.3 Meta Information Row

Display after recipe title:
```
⏱ 25 min   |   👤 2 servings   |   📊 Easy
```
These come directly from the JSON fields `cook_time`, `servings`, `difficulty`.

### 11.4 Tips Section (Optional but Recommended)

At the bottom of the recipe, after all steps:
```
┌─────────────────────────────────────┐
│ 💡  Chef's Tip                      │
│  For extra flavour, marinate the    │
│  chicken in soy sauce for 15 min    │
│  before cooking.                    │
└─────────────────────────────────────┘
```
Background: `#FFF8EE`, border-left: 3px `--accent-secondary`, border-radius 10px.

### 11.5 Scroll Behavior

- Recipe card is a `ScrollView`
- Sticky header is NOT needed (one screen is simple enough)
- Add enough bottom padding (120px) so the last step is never hidden behind the CTA
- Enable momentum scrolling on iOS: `decelerationRate="fast"`

## 12. Accessibility

| Requirement | Implementation |
|---|---|
| All tappable elements have accessible labels | `accessibilityLabel` on every `Pressable` |
| CTA communicates disabled state | `accessibilityState={{ disabled: true }}` when greyed |
| Step numbers are not just visual | `accessibilityLabel="Step 1: ..."` on each step |
| Color is not the only indicator | Disabled state uses both grey color AND reduced opacity |
| Minimum tap target | 44×44px on all interactive elements |
| Text scales with system font size | Use `sp` units, test at 200% font scale |
| Loading state announced | `accessibilityLiveRegion="polite"` on skeleton area |

## 13. Performance Requirements

| Metric | Target |
|---|---|
| First paint (app open) | < 1 second |
| CTA tap → loading state visible | < 300ms |
| API response time (Groq llama3-70b) | Typically 3–8 seconds |
| Recipe rendered after response | < 200ms |
| App bundle size | < 5MB |
| Memory usage | < 80MB |

**Optimizations:**
- Parse JSON once, store in state — don't re-parse on render
- Skeleton animation uses `Animated.loop` with `useNativeDriver: true`
- Avoid re-rendering input field during loading; unmount cleanly
- Tag parsing debounced by 100ms

# Smart Ingredient Recipe Generator

A React Native (Expo) app that generates complete recipes from your available 
ingredients using the Groq API.

## Setup

### Prerequisites
- Node.js 18+
- Expo CLI (`npm install -g expo-cli`)
- A free Groq API key from https://console.groq.com

### Installation

git clone https://github.com/yourusername/smart-ingredient
cd smart-ingredient
npm install

### Configure Environment

Create a `.env` file in the root:

EXPO_PUBLIC_GROQ_API_KEY=your_groq_api_key_here

### Run the App

# iOS
npx expo start --ios

# Android
npx expo start --android

# Expo Go (scan QR code)
npx expo start

## Tech Stack
- React Native + Expo SDK 51
- Groq API (llama3-70b-8192)
- expo-haptics, expo-google-fonts

## AI Integration
Uses Groq's OpenAI-compatible API with a structured JSON prompt. The model 
is instructed to return a minimum of 8 detailed steps per recipe. See 
`services/groqService.js` and `constants/prompts.js`.

## Edge Cases Handled
- Empty input (CTA disabled)
- Single ingredient (warning + proceeds)
- Non-food ingredients (graceful error message)
- Network timeout (15s abort)
- API rate limiting (429 handling)
- Malformed JSON response (fallback display)
